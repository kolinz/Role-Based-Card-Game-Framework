### Prompt 3-2: セッション管理ハンドラ

```
#SDD_Specification_v1.3_full.md の Section 5.3〜5.7 を参照して、
server.js に以下のハンドラを実装してください。

## handleCreateSession(ws, data)
player オブジェクト（seatIndex:0 を含む）:
{
  id, name, categories:[], points:{}, retired:false,
  categorySelected:false, selectedSkillCardIds:[], finished:false, finishRank:null,
  avatarId:null, avatarColorId:null, avatarSelected:false, seatIndex:0
}
送信: { type:'sessionCreated', sessionId, playerId, session }

## handleJoinSession(ws, data)
バリデーション:
  SESSION_NOT_FOUND / SESSION_FULL / GAME_ALREADY_STARTED
seatIndex = session.players.length（push 前）
player オブジェクトの seatIndex を設定してから push
本人: { type:'sessionJoined', playerId, session }
他全員: { type:'playerJoined', session }

## handleSelectAvatar(ws, data)
player.avatarId / player.avatarColorId / player.avatarSelected = true
broadcast: { type:'avatarSelected', playerId, avatarId, avatarColorId, session }

## handleSelectCategory(ws, data) ※ 旧 handleSelectJob
重複チェック: 他プレイヤーが同じ categoryId を選択済みなら CATEGORY_TAKEN エラー
player.categories.push(data.categoryId)
player.categorySelected = true
broadcast: { type:'categorySelected', playerId, categoryId, session }

## handleStartGame(ws, data)
バリデーション（この順番で）:
1. data.playerId !== session.hostPlayerId → NOT_HOST
2. session.players.length < 2           → NOT_ENOUGH_PLAYERS
3. players.some(p => !p.categorySelected)→ NOT_ALL_SELECTED
session.gameStarted = true
broadcast: { type:'gameStarted', session }
```

```
## handleRollDice(ws, data)
better-sqlite3 同期API を使用（コールバック不使用）:

const skillCards      = db.prepare('SELECT * FROM skill_cards').all();
const regularMissions = db.prepare('SELECT * FROM missions WHERE isSpecial = 0').all();
const specialMissions = db.prepare('SELECT * FROM missions WHERE isSpecial = 1').all();

allCards の構築:
- スキルカード: type:'skill', matchesCategories: .split(',').map(Number)
- 通常ミッション: type:'mission'

在庫管理: availableCards.length <= 7 なら usedCardIds をリセット
特殊ミッション: 10% 確率で追加
ランダム抽選（Set で重複防止、diceValue 枚）

broadcast: { type:'diceRolled', diceValue, drawnCards, session }

## handleSelectCard(ws, data)
スキルカードのマッチング判定（matchesCategories と player.categories で比較）:

// 勝利判定（better-sqlite3 同期）
const cats = db.prepare('SELECT * FROM category_cards').all();
const hasWon = player.categories.length > 0 && player.categories.every(cid => {
    const cat = cats.find(c => c.id === cid);
    return cat && (player.points[cid] || 0) >= cat.targetPoints;
});

勝利時: finished=true, finishRank 設定, finishedPlayers に push
全員ゴールチェック: retired でない全員が finished かどうか

本人に ws.send: { type:'cardSelected', matched, alreadySelected, pointsUpdated, playerFinished, finishRank, allFinished, finalRankings, session }
他全員に broadcast: { type:'cardSelectedByOther', playerId, cardType:'skill', session }
allFinished なら全員に broadcast: { type:'gameCompleted', finalRankings, session }
ミッション/特殊カードは broadcast: { ...result, session }
```

```
## よくあるエラーと対処

## Node.js バージョン確認
node --version   → v24.x.x であること
npm --version    → 10.x 以上であること

## better-sqlite3 再ビルド（バージョン不一致の場合）
npm rebuild better-sqlite3

## DB 再初期化
rm game.db && npm run initdb

上記を確認して問題を特定・修正してください。
```

### Prompt D-3: 3D シーン表示問題

```
3D シーンが表示されない（真っ黒、エラー等）場合の診断をしてください。

## Step 1: Three.js の読み込み確認
ブラウザのコンソールで確認:
typeof THREE  → 'object' であること（'undefined' なら CDN 読み込み失敗）

## Step 2: WebGL サポート確認
const canvas = document.createElement('canvas');
const gl = canvas.getContext('webgl2') || canvas.getContext('webgl');
console.log(gl ? 'WebGL OK' : 'WebGL not supported');

## Step 3: #scene-container の確認
document.getElementById('scene-container')  → null でないこと

## Step 4: SceneManager のエラー確認
SceneManager.init() を try/catch で囲んで エラーをコンソールに出力:
try { this.sceneManager.init(players); }
catch (e) { console.error('SceneManager error:', e); }

## Step 5: Three.js r128 の制約確認
- THREE.OrbitControls は r128 CDN に含まれない（使用不可）
- THREE.CapsuleGeometry は r142 以降のため使用不可
- CylinderGeometry / SphereGeometry を使うこと

問題を特定して修正し、3D テーブルとアバターが表示されることを確認してください。
```

### Prompt D-4: WebSocket 接続問題

```
WebSocket が "Disconnected" のままの場合の診断をしてください。

## Step 1: server.js の確認
http.createServer(app) でサーバー作成し、
new WebSocket.Server({ server }) で HTTP サーバーに紐付けていること。
（ポートを直接指定していないこと）

## Step 2: game.js の接続 URL 確認
const protocol = window.location.protocol === 'https:' ? 'wss:' : 'ws:';
const wsUrl = `${protocol}//${window.location.host}`;
ポート番号を明示していないこと（Nginx 経由の場合は不要）

## Step 3: Nginx 設定の確認（本番環境の場合）
以下のヘッダーが設定されているか確認:
proxy_set_header Upgrade $http_upgrade;
proxy_set_header Connection "upgrade";
proxy_http_version 1.1;
proxy_read_timeout 86400;

## Step 4: デバッグログの追加
connect() の ws.onclose に以下を追加（一時的）:
this.ws.onclose = (e) => {
    console.log('Close code:', e.code, 'Reason:', e.reason);
};
```
