### Prompt 3-2: WebSocket ハンドラ関数の実装（createSession / joinSession / selectJob）

```
server.js に以下の3つの WebSocket ハンドラ関数を実装してください。

【handleCreateSession(ws, data)】
処理:
1. sessionId = generateSessionId() でセッションID生成
2. playerId = uuidv4() でプレイヤーID生成
3. session オブジェクト作成:
   {
     id: sessionId,
     hostPlayerId: playerId,
     players: [{
       id: playerId,
       name: data.playerName || 'Player 1',
       jobs: [],           // 選択した職種IDの配列
       points: {},         // { jobId: points } の Map
       retired: false,
       jobSelected: false,
       selectedSkillCardIds: [],
       finished: false,
       finishRank: null
     }],
     maxPlayers: data.maxPlayers || 4,
     currentPlayerIndex: 0,
     gameStarted: false,
     diceValue: null,
     drawnCards: [],
     selectedCardsHistory: [],
     usedCardIds: [],
     chatMessages: [],
     finishedPlayers: [],
     allFinished: false
   }
4. gameSessions.set(sessionId, session)
5. clients Map に sessionId → Map<playerId, ws> を設定
6. currentSessionId と currentPlayerId を更新
7. ws.send で { type: 'sessionCreated', sessionId, playerId, session } を送信

【handleJoinSession(ws, data)】
処理:
1. data.sessionId でセッションを取得。なければエラー送信
2. session.players.length >= session.maxPlayers ならエラー
3. session.gameStarted なら参加不可エラー
4. 新プレイヤーオブジェクトを session.players に追加
5. clients Map に追加
6. currentSessionId, currentPlayerId を更新
7. ws.send で { type: 'sessionJoined', playerId, session }
8. broadcast で他全員に { type: 'playerJoined', session }

【handleSelectJob(ws, data)】
処理:
1. data.sessionId でセッション取得
2. プレイヤーを players から data.playerId で検索
3. data.jobId がすでに他プレイヤーに選択されているかチェック
   → 重複の場合 ws.send でエラー { type: 'error', error: { code: 'JOB_TAKEN' } }
4. player.jobs に data.jobId を追加
5. player.jobSelected = true
6. broadcast で { type: 'jobSelected', playerId: data.playerId, jobId: data.jobId, session }
```

```
6. カード在庫が7枚以下なら全リセット:
   if (availableCards.length <= 7) {

     availableCards = [...allCards];

   if (specialMissions.length > 0 && Math.random() < 0.1) {
     // 特殊ミッション未使用なら availableCards に追加（type: 'special'）
   }

8. ランダム抽選（重複なし）:
   - diceValue 枚だけ Set を使って重複防止しながら抽選
   
9. session.drawnCards に保存
```

### Prompt 3-4: WebSocket ハンドラ関数の実装（selectCard / nextTurn / resign / resetGame）

```
server.js に以下のハンドラ関数を実装してください。

【handleSelectCard(ws, data)】
処理:

1. session.drawnCards から data.cardId でカードを検索
2. 選択履歴に追加:
   session.selectedCardsHistory.push({
     playerId: currentPlayer.id,
     playerName: currentPlayer.name,
     card: card,
     turnNumber: session.selectedCardsHistory.length + 1
   });
3. session.usedCardIds に追加（重複なし）

4. スキルカード (type === 'skill') の場合:
   a. 再選択チェック: player.selectedSkillCardIds.includes(card.id)
      → alreadySelected = true なら ポイント加算なし
   b. 初回選択の場合:
      - card.matchesJobs と player.jobs の積集合でマッチング
      - マッチした jobId ごとに player.points[jobId] += 1
      - player.selectedSkillCardIds に card.id 追加
   c. 勝利判定（db.all で job_cards を取得）:
      - player.jobs の全 jobId について points >= targetPoints か確認
      - 全職種達成 + jobs が空でない + finished でない → player.finished = true
      - finishRank = session.finishedPlayers.length + 1
      - session.finishedPlayers に追加
   d. 全員ゴールチェック:
      - retired でないプレイヤー全員が finished か確認
      - 全員達成なら session.allFinished = true
      - finalRankings を finishRank 順にソートして生成
   e. ws.send で result（matched, alreadySelected, pointsUpdated, playerFinished, finishRank, allFinished, finalRankings, session）
   f. broadcast で他プレイヤーに { type: 'cardSelectedByOther', session }
   g. allFinished なら broadcast で { type: 'gameCompleted', finalRankings, session }

5. ミッション/特殊カードの場合:
   - broadcast で { type: 'cardSelected', card, session }

【handleNextTurn(ws, data)】
- currentPlayerIndex を (index + 1) % players.length でインクリメント
- retired プレイヤーをスキップ（ループして次の active プレイヤーを探す）
- session.drawnCards をクリア
- broadcast で { type: 'nextTurn', session }

【handleResign(ws, data)】
処理（特殊ミッション: 退職）:
1. 退職するプレイヤー (data.playerId) と引き継ぐプレイヤー (data.targetPlayerId) を取得
2. 引き継ぎプレイヤーの jobs に退職プレイヤーの jobs を追加（兼任）
3. 新しく追加された jobId の points を 0 で初期化
4. 退職プレイヤー: retired = true, jobs = [], points = {}
5. broadcast で { type: 'playerRetired', retiredPlayerId, targetPlayerId, session }

【handleResetGame(ws, data)】
- session の gameStarted, drawnCards, selectedCardsHistory, usedCardIds, finishedPlayers, allFinished をリセット
- 全プレイヤーの jobs, points, retired, jobSelected, selectedSkillCardIds, finished, finishRank をリセット
- session.currentPlayerIndex = 0
- broadcast で { type: 'gameReset', session }
```

```

app.use(express.static(__dirname));

- body: { username, password }
- 一致: { ok: true, token: uuidv4(), expiresIn: TOKEN_EXPIRY }

【管理API（認証必須: requireAdmin）】

- 必須: name_en, name_ja, targetPoints（なければ 400）
- targetPoints は数値バリデーション
- INSERT 後 { ok: true, id: this.lastID }

PUT /api/admin/jobs/:id
- 既存レコードを更新
- { ok: true }

- { ok: true }

同様に skill_cards 用:
POST/PUT/DELETE /api/admin/skills

missions 用:
POST/PUT/DELETE /api/admin/missions

【CSV Import API】
POST /api/admin/import/:type
- type: 'jobs' | 'skills' | 'missions'
- body: { csvData: [...], preview: boolean }
- preview=true のときはDBへ書き込まず、件数だけ返す
- preview=false のときは upsert（id があれば UPDATE, なければ INSERT）
```

```
【connect() メソッド】
connect() {
  const protocol = window.location.protocol === 'https:' ? 'wss:' : 'ws:';
  const wsUrl = `${protocol}//${window.location.host}`;
  this.ws = new WebSocket(wsUrl);

  this.ws.onopen = () => {
    this.connected = true;
    this.updateConnectionIndicator();
    this.render();
  };

  this.ws.onclose = () => {
    this.connected = false;
    this.updateConnectionIndicator();
    this.render();
    setTimeout(() => this.connect(), 3000);  // 自動再接続
  };

  this.ws.onmessage = (event) => {
    const data = JSON.parse(event.data);
    this.handleServerMessage(data);
  };

  this.ws.onerror = (err) => {
    console.error('WebSocket error:', err);
  };
}
```

### Prompt D-1: WebSocket 接続デバッグ

```
WebSocket 接続が "Disconnected" のまま解決しない場合のデバッグをしてください。

【現象】
ブラウザのコンソールに以下が表示される:
WebSocket connection to 'wss://your-domain.com/' failed

【確認事項】
1. server.js で WebSocket サーバーが正しく起動しているか
2. Nginx の proxy_set_header Upgrade $http_upgrade が設定されているか
3. ファイアウォールで port 3000 が開いているか
4. HTTPS 環境で wss:// を使っているか（ws:// ではなく）

【デバッグ用コード】
game.js の connect() メソッドに以下のログを追加してください:
- ws.onopen: "WebSocket connected to: " + wsUrl
- ws.onclose: "WebSocket closed. Code: " + event.code + " Reason: " + event.reason
- ws.onerror: "WebSocket error: " + event

上記を確認した上で、問題を特定して修正してください。
```
